# Прототип «Единый ИИ-шлюз для госсектора»

Кликабельный high-fidelity прототип веб-приложения для демонстрации стейкхолдерам (руководство, ИБ, заказчики-ведомства).

## Запуск

```bash
cd ai-gateway-prototype
npm install
npm run dev
```

Откройте в браузере адрес, указанный в терминале (обычно http://localhost:5173).

## Публикация в интернет

Соберите проект и выберите один из вариантов.

### 1. Vercel (рекомендуется)

Бесплатно, без регистрации для разового деплоя через CLI:

```bash
npm run build
npx vercel --yes
```

Следуйте подсказкам в терминале. В конце получите ссылку вида `https://ai-gateway-prototype-xxx.vercel.app`.  
Для постоянного деплоя: зарегистрируйтесь на [vercel.com](https://vercel.com), подключите репозиторий GitHub — каждый push будет автоматически выкладывать новую версию.

### 2. Netlify

Через веб-интерфейс: зайдите на [app.netlify.com](https://app.netlify.com) → «Add new site» → «Deploy manually» и перетащите папку **`dist`** (после `npm run build`) в окно браузера.

Через CLI:

```bash
npm run build
npx netlify deploy --prod --dir=dist
```

### 3. GitHub Pages

Положите проект в репозиторий на GitHub. В настройках репозитория: **Settings → Pages → Source**: выберите **GitHub Actions**. В репозитории создайте файл `.github/workflows/deploy.yml` по [документации Vite](https://vitejs.dev/guide/static-deploy.html#github-pages) для деплоя из ветки. После успешного запуска workflow сайт будет доступен по адресу `https://<username>.github.io/<repo>/`. В `vite.config.ts` нужно задать `base: '/<repo>/'`.

## Возможности прототипа

- **Навигация** — левое меню: Dashboard, Playground, Каталог моделей, Маршрутизация, Шаблоны, DLP, Аудит, Usage, Администрирование, Интеграции, О платформе.
- **Роли (демо)** — переключатель в шапке: Сотрудник ведомства, Администратор ведомства, Офицер ИБ, Админ платформы. Для Platform Admin в Администрировании отображается блок «Провайдеры».
- **Каталог моделей** — фильтры по провайдеру и «только для госсектора», карточки моделей, сравнение, кнопка «Запросить подключение» → переход в Маршрутизацию (статус «Ожидает согласования ИБ»).
- **Playground** — выбор проекта, модели, режима; ввод запроса; индикатор комплаенса (OK/Blocked/Masked); кнопки «Сохранить как шаблон», «Отправить в аудит», «Экспорт ответа».
- **Маршрутизация и политики** — таблицы политик и лимитов, статусы (черновик / на согласовании / активно).
- **DLP и безопасность** — правила DLP, таблица инцидентов, ссылка на Аудит.
- **Аудит** — таблица запросов, клик по строке → детали события (превью входа/выхода, сработавшие правила).
- **Usage и биллинг** — использование по проектам, лимиты, уведомления 80%, тарифы, кнопка «Скачать отчёт» (mock CSV).
- **Тема** — переключатель «Светлая / Тёмная» в шапке.

## Технологии

- React 18, TypeScript, Vite, React Router.
- Мок-данные в `src/data/` (модели, провайдеры, аудит, usage, DLP, политики, шаблоны, проекты, интеграции).
- Стили: CSS-переменные, светлая/тёмная тема, desktop-first.

## Ограничения

Реальная авторизация (ЕСИА/AD), вызовы LLM (LiteLLM/vLLM), DLP-движки и платёжные системы не реализованы — только UI и мок-данные.
